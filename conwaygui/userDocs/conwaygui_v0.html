Introduction

L'applicazione ConwayLife25 è un'applicazione Java che simula l'evoluzione di un insieme di celle in un ambiente bidimensionale. Questo documento descrive il progetto di realizzazione di una pagina HTML che funga da dispositivo di input-output evoluto per l'applicazione ConwayLife25 in Java.

Requirements

Goal: creare una gui che permetta il controllo dell'applicazione ConwayLife25 in Java.

Requirement analysis

Per una gui serve
- una griglia che permetta di visualizzare lo stato delle celle e di modificarlo.
- un pulsante per far evolvere le celle (START).
- un pulsante per fermare l'evoluzione (STOP).
- un pulsante per uscire dall'applicazione (EXIT).
Ci sono due tipologie di interlocutori del nostro sistema:
  1. Possessore del gioco
  2. Spettatore del gioco

Problem analysis

Bisogna far interagire la pagina HTML con l'applicazione Java ConwayLife25 in backend.
L'evoluzioni del gioco deve essere visualizzata in tempo reale sulla pagina HTML. La comunicazione tra la pagina HTML e l'applicazione Java deve essere bidirezionale e avverrà tramite un protocollo di comunicazione (es. http o websocket).
Fase di inizializzazione: Quando si clicca su una cella della griglia, la cella cambia stato (viva/morta). Questo cambiamento verrà visualizzato tramite un cambio di colore della cella. L'interazione tra la pagina e la cella/applicazione può essere realizzata tramite websocket o http.
Quando si preme il pulsante START: il gioco inizia e le celle evolvono. Lato applicazione il controller azionerà l'esecuzione tramite il metodo play().
Quando si preme il pulsante STOP: il gioco si ferma e le celle non evolvono più. In backend si è predisposto di un metodo di stop nel lifeColtroller.
Quando si preme il pulsante EXIT: l'applicazione si chiude (nel caso di utente possessore del gioco) oppure si disconnette (nel caso di spettatore del gioco).

Organizzazione del processo sviluppo

Abbiamo utilizzato il framework spring boot per l'organizzazione dei package e per la Dependency Injection.
La business logic può essere integrata all'interno dell'applicazione in 2 modi diversi:
  1. Innestandola direttamente all'interno del progetto Conwaygui
  2. In maniera impicita, creando un progetto a parte e integrandolo come dipendenza (jar)
In questa fase iniziale si è optato per la prima soluzione, in quanto l'applicazione può ancora variare.
